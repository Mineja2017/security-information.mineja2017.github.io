# Summary

## € Schneier 2015: Applied Cryptography
### 1. One way Functions
They are key in publick-key criptography. The fundamental here is that One way functions are easy to compute and incredibly hard to reverse.

For example:

- Take numbers 2,3 and 5 as teh thing you want to compute with the One-Way function.
- Let's consdier that our functions is just the sum of the numbers given. 
- The result in this case would be 10.
  
Just by having 10 as data avaliable, it would be impossible to know which numebrs were originally given (1+9, 8+2, ...)
As an interesting fact, there is no mathematical proof of the existence of One-way functions.

### 2. Trapdoor One-Way Functions
These are a special subset of one-way functions that include a "shortcut."

- They are also hard to reverse unless you have access to a specific piece of secret or trapdoor.
- This is the base of Public-Key Cryptography. It allows anyone to encrypt a message that is only feasable to reverse to the person that holds the private key.

### 3. One-Way Hash Functions
It is a One-way function that takes a variable lenght input and outputs a fixed lenght text.

Key properties:
- **Collision-Free**: It is extremely difficult to find two different inputs that produce the same hash.
- **High Sensitivity**: Changing just one bit of the input changes about half of the bits in the output.

It's primary use is to provide integrity to a message.

### 4. Message Authentication Codes (MACs)
It is a One-Way function that requieres a code or secret key to compute tha hash.

- **Difference from Hash Functions**: anyone can compute a standard hash, only someone with the specific key can verify a MAC

It's primary use is to provide authenticity.

# a. Install hashcat and test

If you have kali linux installed hashcat should be already installed. If not, you can use the following command to install it:
```bash
sudo apt install hashcat hashid
```
![install hashcat](https://raw.githubusercontent.com/Mineja2017/security-information.mineja2017.github.io/refs/heads/main/assets/h6/install_hash.png)

As explained in the article we can use hashid to identify the type of hash we are working with using the -m option:
```bash
hashid -m <hash>
```
![hashid](https://raw.githubusercontent.com/Mineja2017/security-information.mineja2017.github.io/refs/heads/main/assets/h6/hashid.png)

In this case we can see that the hash is probably a md5 hash which is option 0 in hashcat using the following commamnd we will use hashcat to decrypt using rockyou dictionary and the fastes way possible.

```bash
hashcat -m 0 6b1628b016dff46e6fa35684be6acc96 -a 0 /usr/share/wordlists/rockyou.txt -w 4 -O -S
```
Where:
- -m 0: md5 hash
- -a 0: dictionary attack
- -w 4: With this option hashcat will use the maximun amount of GPU
- -O: We ausume that the maximun lenght of the password is 32 characters (optimized for the kernel)
- -S: MD5 is realy fast algorithm so we activate the slow candidates flag.
  
![hashcat](https://raw.githubusercontent.com/Mineja2017/security-information.mineja2017.github.io/refs/heads/main/assets/h6/hashcat.png)

# n. Crack the hash of a zip file using john the ripper
If you are using kali linux john the ripper should be already installed. If not, you can use the following command to install it:
```bash
sudo apt install john
```
Now he create a secret file with the following content:
```bash
echo "secret_file" >> secret.txt
```
And then we create a zip file with the following command:
```bash
zip -e secret.zip secret.txt
```
And finally we can use john the ripper to crack the hash of the zip file:
```bash
zip2john secret.zip >> hash.txt ; john hash.txt
```
